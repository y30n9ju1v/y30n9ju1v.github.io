## 1. Introduction: 단안 3D 탐지의 한계

**요약**

자율주행 시스템은 주변 물체의 3D 위치와 크기를 정확히 파악해야 합니다. LiDAR 기반 방법은 이미 높은 성능을 보이지만, LiDAR 장비는 비싸고 먼 거리에서는 포인트 클라우드가 희박해지는 문제가 있습니다. 반면 단안 카메라 이미지만을 활용하는 방법은 LiDAR 대비 10%도 안 되는 성능을 보였습니다.

그 이유는 **원근 투영(perspective projection)** 때문입니다. 이미지에서 같은 물체라도 카메라로부터의 거리에 따라 크기가 달라 보이고, 3D 세계의 실제 거리를 직접 알 수 없습니다. 이 논문은 이 문제를 해결하기 위해 이미지 피처를 **직교 조감도 공간(orthographic birds-eye-view)**으로 변환하는 방법을 제안합니다.

**핵심 개념**

- **원근 투영의 문제점**: 같은 물체도 멀리 있으면 작게 보이고 가까이 있으면 크게 보임 → 스케일 불일치
- **조감도(BEV) 표현의 장점**: 스케일이 균일하고, 물체 간 실제 거리가 의미 있으며, 시점 변화에 덜 민감함
- **핵심 아이디어**: 가능한 한 많은 추론을 이미지 공간이 아닌 직교 3D 공간에서 수행해야 한다

## 2. Related Work

**요약**

기존 3D 탐지 방법들은 크게 세 가지로 나뉩니다.

1. **LiDAR 기반 방법**: 포인트 클라우드를 직접 처리하거나(PointNet 계열), 이미지에 투영하거나, 조감도 맵으로 변환. 이 논문의 OFT는 이미지만으로 이런 조감도 맵을 암묵적으로 구성하려는 시도임
2. **이미지 기반 3D 탐지**: 2D 박스에서 3D 파라미터를 회귀하거나, 3D 템플릿을 피팅. 각 제안을 독립적으로 처리하기 때문에 장면 전체를 공동으로 추론하지 못하는 한계
3. **인테그럴 이미지**: 빠른 영역 평균 풀링을 위한 고전적 기법. 이 논문은 인테그럴 이미지를 역전파 가능한 end-to-end 딥러닝 아키텍처에 통합한 최초 사례 중 하나

**핵심 개념**

- **Mono3D**: 3D 공간에 조밀하게 제안을 만들고 이미지 피처로 스코어링 → 공간적 맥락 추론 불가
- **AVOD / MV3D**: LiDAR + 이미지 융합 → OFT와 유사한 피처 집계 수행하지만 LiDAR 필요
- **인테그럴 이미지(Integral Image)**: Viola-Jones 이후 물체 탐지의 핵심 요소. OFT에서 대용량 영역 풀링을 가능하게 함

## 3. 3D Object Detection Architecture

**요약**

OFT-Net의 전체 파이프라인은 5단계로 구성됩니다:

1. **ResNet-18 프론트엔드**: 입력 이미지에서 다중 스케일 피처 맵 추출
2. **Orthographic Feature Transform (OFT)**: 이미지 피처를 3D 복셀 공간을 거쳐 조감도 직교 피처 맵으로 변환
3. **Topdown Network**: 조감도 공간에서 ResNet 기반으로 3D 장면 추론
4. **Output Heads**: 각 지점에서 신뢰도, 위치 오프셋, 크기 오프셋, 방향 예측
5. **Non-Maximum Suppression**: 신뢰도 맵에서 피크를 찾아 최종 박스 생성

### 3.1 피처 추출 (Feature Extraction)

ResNet-18의 마지막 세 다운샘플링 레이어 직전에서 피처를 추출하여 원본 이미지의 1/8, 1/16, 1/32 해상도의 다중 스케일 피처 맵 `{f^s}`를 생성합니다. 1×1 컨볼루션으로 공통 피처 크기 256으로 통일합니다.

### 3.2 Orthographic Feature Transform (OFT)

OFT의 핵심은 이미지 피처 `f(u,v)`를 3D 복셀 피처 `g(x,y,z)`로 변환한 뒤, 이를 수직 방향으로 합쳐 2D 조감도 피처 `h(x,z)`를 만드는 것입니다.

**복셀의 이미지 투영 좌표 계산:**

$$u_1 = f\frac{x - 0.5r}{z + 0.5\frac{x}{|x|}r} + c_u, \quad u_2 = f\frac{x + 0.5r}{z - 0.5\frac{x}{|x|}r} + c_u$$

$$v_1 = f\frac{y - 0.5r}{z + 0.5\frac{y}{|y|}r} + c_v, \quad v_2 = f\frac{y + 0.5r}{z - 0.5\frac{y}{|y|}r} + c_v$$

**수식 설명**

이 수식은 3D 공간의 복셀(작은 정육면체)이 카메라 이미지에서 어느 영역에 해당하는지 계산합니다:

- **$f$**: 카메라 초점 거리(focal length). 카메라의 줌 정도를 나타냄
- **$c_u, c_v$**: 카메라의 주점(principal point). 이미지의 중심에 해당하는 픽셀 좌표
- **$r$**: 복셀의 크기(예: 0.5m)
- **$(x, y, z)$**: 3D 세계 좌표계에서 복셀의 중심 위치
- **$(u_1, v_1), (u_2, v_2)$**: 복셀이 이미지에 투영된 사각형의 좌상단/우하단 픽셀 좌표
- **직관**: 멀리 있는 복셀($z$ 큼)은 이미지에서 작은 영역에 투영되고, 가까이 있는 복셀은 큰 영역에 투영됨

**이미지 피처의 복셀 집계:**

$$\mathbf{g}(x, y, z) = \frac{1}{(u_2 - u_1)(v_2 - v_1)} \sum_{u=u_1}^{u_2} \sum_{v=v_1}^{v_2} \mathbf{f}(u, v)$$

**수식 설명**

이 수식은 복셀에 해당하는 이미지 영역의 피처를 평균 풀링하여 복셀 피처를 구합니다:

- **$\mathbf{g}(x, y, z)$**: 3D 위치 $(x,y,z)$의 복셀 피처 벡터
- **$\mathbf{f}(u, v)$**: 이미지 픽셀 $(u,v)$의 피처 벡터
- **분모 $(u_2-u_1)(v_2-v_1)$**: 투영 영역의 픽셀 수 → 평균을 내기 위해 나눔
- **직관**: 복셀이 이미지의 어느 부분에 해당하는지 찾아서 그 부분의 피처를 평균 낸 것

**3D → 2D 조감도 피처 변환:**

$$\mathbf{h}(x, z) = \sum_{y=y_0}^{y_0+H} W(y)\mathbf{g}(x, y, z)$$

**수식 설명**

3D 복셀 피처를 수직 방향으로 합산하여 2D 조감도 피처를 만듭니다:

- **$\mathbf{h}(x, z)$**: 조감도 위치 $(x,z)$의 피처 (최종 2D BEV 피처 맵)
- **$W(y) \in \mathbb{R}^{n \times n}$**: 높이 $y$에 따라 다르게 적용되는 학습 가능한 가중치 행렬
- **$y_0$**: 지면으로부터 카메라까지의 높이 기준점
- **$H$**: 수직 방향 복셀 범위
- **직관**: 단순히 더하는 게 아니라 높이별로 서로 다른 가중치를 학습함으로써, 물체의 수직 구성(높이, 지면 위 위치) 정보를 보존

### 3.2.1 인테그럴 이미지를 활용한 빠른 평균 풀링

약 15만 개의 복셀을 처리해야 하는 문제를 해결하기 위해 인테그럴 피처 맵(Integral Feature Map)을 사용합니다.

**인테그럴 피처 맵 구성:**

$$\mathbf{F}(u, v) = \mathbf{f}(u, v) + \mathbf{F}(u-1, v) + \mathbf{F}(u, v-1) - \mathbf{F}(u-1, v-1)$$

**수식 설명**

누적합 형태의 자료구조를 만드는 점화식입니다:

- **$\mathbf{F}(u, v)$**: 좌상단 $(0,0)$부터 현재 픽셀 $(u,v)$까지의 모든 피처의 합
- **$\mathbf{f}(u, v)$**: 현재 픽셀의 피처
- **직관**: 전처리를 한 번 해두면 임의의 사각형 영역 합을 O(1) 시간에 계산 가능

**인테그럴 이미지로 복셀 피처 계산:**

$$\mathbf{g}(x, y, z) = \frac{\mathbf{F}(u_1, v_1) + \mathbf{F}(u_2, v_2) - \mathbf{F}(u_1, v_2) - \mathbf{F}(u_2, v_1)}{(u_2 - u_1)(v_2 - v_1)}$$

**수식 설명**

네 꼭짓점 값의 조합으로 사각형 영역의 합을 O(1)에 계산합니다:

- **직관**: 전체 합에서 필요한 사각형 밖의 부분을 빼고 중복 제거
- **핵심 장점**: 복셀 크기(영역 크기)와 무관하게 복잡도가 일정함 → 먼 곳의 복셀(크게 투영됨)도 가까운 곳 복셀(작게 투영됨)과 같은 비용으로 처리
- **역전파 가능**: 원래 피처 맵 f에 대해 미분 가능 → end-to-end 학습 가능

### 3.3 Topdown Network

조감도 피처 맵 `h`에 ResNet 스타일의 스킵 연결을 가진 16층 컨볼루션 네트워크를 적용합니다. 모든 처리가 지면 위 위치에 대해 변환 불변(translation-invariant)하므로, 카메라에서 멀리 있는 물체와 가까이 있는 물체를 동일하게 처리합니다.

### 3.4 신뢰도 맵 예측 (Confidence Map)

$$S(x, z) = \max_i \exp\left(-\frac{(x_i - x)^2 + (z_i - z)^2}{2\sigma^2}\right)$$

**수식 설명**

각 지면 위치 $(x,z)$에서 물체가 있을 확률을 가우시안으로 표현합니다:

- **$S(x, z)$**: 위치 $(x,z)$의 신뢰도 (0~1)
- **$(x_i, z_i)$**: $i$번째 실제 물체의 지면 중심 좌표
- **$\sigma$**: 가우시안 너비 (피크의 퍼짐 정도)
- **$\max_i$**: 여러 물체 중 가장 가까운 물체의 가우시안 값을 사용
- **직관**: 물체 중심에서 가까울수록 1에 가깝고, 멀수록 0에 가까운 매끄러운 확률 맵

손실 함수는 $\ell_1$ 손실 사용. 음성 위치(물체 없는 곳)가 압도적으로 많기 때문에, $S < 0.05$ 위치의 손실에 $10^{-2}$ 가중치를 곱해 불균형 문제 해소.

### 3.5 위치·크기·방향 예측

각 위치에서 세 가지 추가 출력을 예측합니다:

**위치 오프셋:**

$$\Delta_{pos}(x, z) = \left[\frac{x_i - x}{\sigma} \quad \frac{y_i - y_0}{\sigma} \quad \frac{z_i - z}{\sigma}\right]^\top$$

**크기 오프셋 (로그 스케일):**

$$\Delta_{dim}(x, z) = \left[\log\frac{w_i}{\bar{w}} \quad \log\frac{h_i}{\bar{h}} \quad \log\frac{l_i}{\bar{l}}\right]^\top$$

**수식 설명**

- **$w_i, h_i, l_i$**: 예측 물체의 너비/높이/길이
- **$\bar{w}, \bar{h}, \bar{l}$**: 해당 클래스 전체의 평균 너비/높이/길이
- **로그 스케일**: 절대값 대신 비율로 예측 → 작은 물체와 큰 물체를 균형 있게 학습

**방향 벡터:**

$$\Delta_{ang}(x, z) = \left[\sin\theta_i \quad \cos\theta_i\right]^\top$$

**수식 설명**

- **$\theta_i$**: 물체의 y축 회전각 (조감도에서 차량이 향하는 방향)
- **$\sin\theta, \cos\theta$**: 각도 대신 사인/코사인 쌍으로 예측하면 각도의 불연속성(0°와 360° 동치) 문제 해결
- **직관**: 조감도 공간에서 작동하므로 원근 효과를 보정하는 별도의 "관측 각도(observation angle)"가 필요 없음

### 3.6 Non-Maximum Suppression

신뢰도 맵 $S$에서 가우시안 스무딩 후 로컬 최댓값을 탐지하여 최종 박스를 생성합니다. 조감도에서는 물체들이 같은 공간을 점유할 수 없으므로 피크들이 자연스럽게 분리됩니다.

## 4. Experiments

**요약**

### 실험 설정

- **프론트엔드**: ResNet-18 (얕게 유지 → 3D 추론에 집중)
- **복셀 그리드**: 80m × 4m × 80m, 해상도 0.5m
- **Topdown Network**: 다운샘플링 없는 16층 ResNet
- **정규화**: Batch Normalization → Group Normalization으로 교체 (소배치 학습에 유리)
- **데이터셋**: KITTI 3D Object Detection Benchmark (학습 3712장, 검증 3769장)
- **데이터 증강**: 랜덤 크롭, 스케일, 수평 플립 + 카메라 파라미터 보정
- **학습**: SGD 600 에폭, 배치 크기 8, 모멘텀 0.9, 학습률 $10^{-7}$

### KITTI 테스트 벤치마크 결과 (Car 클래스)

| 방법 | 모달리티 | AP3D Easy | AP3D Mod | AP3D Hard | APBEV Easy | APBEV Mod | APBEV Hard |
|------|----------|-----------|----------|-----------|------------|-----------|------------|
| 3D-SSMFCNN | Mono | 2.28 | 2.39 | 1.52 | 3.66 | 3.19 | 3.45 |
| **OFT-Net (Ours)** | **Mono** | **2.50** | **3.28** | **2.27** | **9.50** | **7.99** | **7.51** |

### KITTI 검증 세트 결과

| 방법 | 모달리티 | AP3D Easy | AP3D Mod | AP3D Hard | APBEV Easy | APBEV Mod | APBEV Hard |
|------|----------|-----------|----------|-----------|------------|-----------|------------|
| 3DOP | Stereo | 6.55 | 5.07 | 4.10 | 12.63 | 9.49 | 7.59 |
| Mono3D | Mono | 2.53 | 2.31 | 2.31 | 5.22 | 5.19 | 4.13 |
| **OFT-Net (Ours)** | **Mono** | **4.07** | **3.27** | **3.29** | **11.06** | **8.79** | **8.91** |

OFT-Net은 단안 방법 중 최고 성능을 달성하고, 일부 지표에서 스테레오 방법인 3DOP를 능가합니다.

### 소거 실험 (Ablation Study)

Topdown Network 레이어 수를 줄일수록 성능이 급격히 저하됩니다. ResNet-18(얕은 백본) + 16층 Topdown이 ResNet-34(깊은 백본) + Topdown 없음보다 파라미터 수가 비슷함에도 훨씬 높은 성능을 보입니다. → **3D 추론의 중요성 입증**

## 5. Discussion

**요약**

### 거리별 성능

OFT-Net은 멀리 있는 물체(30m 이상)에서도 Mono3D보다 훨씬 느린 성능 저하를 보입니다. 조감도 공간에서 처리하기 때문에 카메라에서 멀더라도 같은 해상도로 처리되기 때문입니다.

### 학습 과정에서의 신뢰도 맵 진화

- **초기(Epoch 10~30)**: 물체가 있는 방향(투영선)을 빠르게 파악하지만, 깊이(depth) 방향으로 불확실성이 큼 → 길게 늘어진 블러 형태
- **후기(Epoch 70~90)**: 깊이 불확실성이 점차 해소되어 물체 중심 근처에 집중된 날카로운 피크 형성
- **관찰**: 멀리 있는 물체일수록 깊이 불확실성이 더 크게 남음 → 스테레오 깊이 추정에서 거리 오차가 거리 제곱에 비례하는 것과 일치

## 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **OFT (Orthographic Feature Transform)** | 이미지 피처를 3D 복셀을 거쳐 조감도 직교 피처로 변환하는 미분 가능한 변환 |
| **Voxel Feature Map** | 3D 공간을 균일한 격자로 나눈 복셀에 이미지 피처를 집계한 중간 표현 |
| **Orthographic Feature Map** | 복셀 피처를 수직 방향으로 가중합산한 2D 조감도 피처 맵 |
| **Integral Image** | 임의 사각형 영역의 합을 O(1)에 계산하는 자료구조. 역전파 가능하도록 구현 |
| **Topdown Network** | 조감도 피처에서 3D 장면을 추론하는 ResNet 기반 서브네트워크 |
| **Confidence Map** | 각 지면 위치에서 물체 존재 확률을 가우시안으로 표현한 맵 |
| **Group Normalization** | 소배치 크기에서도 안정적인 학습을 위해 Batch Normalization 대체 |
| **BEV (Birds-Eye-View)** | 위에서 내려다본 조감도 표현. 스케일 균일, 거리 의미 있음 |

## 결론 및 시사점

이 논문의 핵심 기여는 **"이미지 피처를 조감도 공간으로 변환하면 단안 카메라로도 효과적인 3D 물체 탐지가 가능하다"**는 것을 실증적으로 보인 것입니다.

**주요 성과**:
- 단안 방법 중 KITTI 벤치마크 SOTA 달성
- 일부 지표에서 스테레오 방법(3DOP) 능가
- 먼 거리 물체 탐지에서 특히 강점

**실무적 시사점**:
- BEV 공간에서의 추론은 이후 BEVFusion, BEVDet, LSS(Lift-Splat-Shoot) 등 자율주행 인식 연구의 방향성을 제시
- 명시적 깊이 추정 없이도 암묵적으로 3D 구조를 학습할 수 있음
- 인테그럴 이미지를 역전파 가능한 풀링으로 활용하는 아이디어는 효율적 특징 집계의 좋은 사례

**한계**:
- 지면 평면(ground plane) 가정에 의존 → 경사로, 언덕 등 비평탄 지형에서 취약
- 전체 성능 수준(AP3D Moderate ~3%)은 LiDAR 방법과 여전히 큰 격차 존재
- 단일 카메라 기준이므로 다중 카메라로 확장 시 별도 고려 필요
