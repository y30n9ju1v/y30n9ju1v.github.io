## 개요

- **저자**: Bernhard Kerbl, Georgios Kopanas, Thomas Leimkühler, George Drettakis
- **발행년도**: 2023 (August)
- **출처**: ACM Transactions on Graphics (SIGGRAPH 2023)
- **논문 아이디**: arXiv:2308.04079v1

이 논문은 **3D Gaussian Splatting**이라는 혁신적인 방법을 제시하여, 신경 방사장(Neural Radiance Field, NeRF)의 높은 품질을 유지하면서도 실시간 렌더링(≥30fps @ 1080p)을 가능하게 합니다. 기존의 NeRF 방식은 훈련 속도와 렌더링 속도 사이에서 품질 손실이 불가피했으나, 이 연구는 그 균형을 깨뜨렸습니다.

## 주요 혁신

**성능 비교**
- InstantNGP: 7분 훈련, 9.2fps (PSNR: 22.1)
- Plenoxels: 26분 훈련, 8.2fps (PSNR: 21.9)
- Mip-NeRF360: 48시간 훈련, 0.071fps (PSNR: 24.3)
- **제안 방법**: 51분 훈련, 93fps (PSNR: 25.2) - 최고 품질 달성

## 주요 내용

### 1. 기존 방법의 한계

**신경 방사장(NeRF) 방식의 문제**
- 연속적인 장면 표현을 위해 MLP 신경망 사용
- 체적 광선 추적(volumetric ray-marching)으로 인해 렌더링이 느림
- 확률적 샘플링으로 인한 노이즈 발생

**최근 빠른 방법들의 한계**
- 속도를 위해 품질을 포기
- 메시(mesh)나 점(point) 기반 표현은 명시적이지만 최적화 어려움
- 그리드 기반 방식(voxel, hash grid)도 렌더링 속도와 품질의 트레이드오프 존재

### 2. 제안 방법의 핵심 아이디어

#### 3D 가우시안 표현 (3D Gaussian Representation)

3D 가우시안은 다음과 같이 정의됩니다:

$$G(\mathbf{x}) = \exp\left(-\frac{1}{2}(\mathbf{x} - \boldsymbol{\mu})^T \Sigma^{-1}(\mathbf{x} - \boldsymbol{\mu})\right)$$

**수식 설명**
이 수식은 3차원 공간에서 종 모양의 분포를 나타냅니다:
- **$G(\mathbf{x})$**: 위치 $\mathbf{x}$에서의 가우시안 값 (중심에서 멀어질수록 작아짐)
- **$\mathbf{x}$**: 3D 공간의 좌표 (x, y, z)
- **$\boldsymbol{\mu}$**: 가우시안의 중심 위치 (mean) - 초점 역할
- **$\Sigma$**: 공분산 행렬(covariance matrix) - 가우시안의 크기와 형태 결정
- **$\Sigma^{-1}$**: 공분산 행렬의 역행렬 - 수식을 계산하기 위한 수학적 도구
- **지수 함수**: 중심에서 먼 거리의 점들은 지수적으로 빠르게 0에 가까워짐

**직관**: 연기가 중심에서 짙고 멀어질수록 옅어지는 모양을 생각하면 됩니다.

**장점**
- 카메라 캘리브레이션에서 생성된 희소 점(sparse point)들로부터 시작
- 연속적인 볼륨 방사장의 좋은 성질 유지
- 빈 공간에서의 불필요한 계산 제거

### 3. 최적화 및 밀도 제어

#### 교대 최적화/밀도 제어 (Interleaved Optimization/Density Control)

장면을 최적으로 표현하기 위해 다음을 수행합니다:

1. **이방성 공분산 최적화**: 장면의 정확한 표현을 위해 3D 가우시안의 방향성 있는 공분산 최적화
2. **적응적 밀도 제어**:
   - 높은 기울기 영역에서는 가우시안 분할(split)
   - 낮은 기울기 영역에서는 가우시안 제거(prune)
   - 투명도(opacity)에 따른 동적 조정

**손실 함수 (Loss Function)**

$$\mathcal{L} = (1-\lambda) \cdot L_1(\hat{C}, C) + \lambda \cdot L_{SSIM}(\hat{C}, C)$$

**수식 설명**
이 수식은 훈련 중 모델이 정답에 얼마나 가까운지 측정하는 방법입니다:
- **$\mathcal{L}$**: 총 손실값 (작을수록 좋음)
- **$L_1(\hat{C}, C)$**: 색상 오차 - 예측 색상($\hat{C}$)과 실제 색상($C$)의 픽셀별 차이
  - 예: 예측 RGB(255, 0, 0)와 실제 RGB(254, 1, 0)의 차이 = 1+1+0 = 2
- **$L_{SSIM}(\hat{C}, C)$**: 구조 유사성 손실 - 사람 눈이 인지하는 이미지 품질 오차
  - 세부 구조, 엣지, 질감 등을 함께 고려
- **$\lambda$**: 가중치 파라미터 (보통 0.2) - 두 손실의 중요도 조정
  - $\lambda = 0.2$이면 색상 오차 80% + 구조 유사성 20% 반영

**직관**: 두 가지 다른 방식으로 오차를 측정해서 더 정확한 훈련이 가능합니다.

### 4. 고속 렌더링 알고리즘

#### 가시성 인식 렌더링 (Visibility-Aware Rendering)

핵심은 **타일 기반 스플래팅(tile-based splatting)**입니다:

$$C(\mathbf{p}) = \sum_{i=1}^{n} T_i c_i \alpha_i, \quad T_i = \prod_{j=1}^{i-1}(1-\alpha_j)$$

**수식 설명**
이 수식은 여러 가우시안을 투명도를 고려하여 합치는 과정입니다:
- **$C(\mathbf{p})$**: 픽셀 p의 최종 색상 (화면에 표시되는 색)
- **$n$**: 이 픽셀에 영향을 주는 가우시안의 개수
- **$c_i$**: i번째 가우시안의 색상 (RGB 값)
- **$\alpha_i$**: i번째 가우시안의 투명도 (0 = 완전 투명, 1 = 완전 불투명)
- **$T_i$**: i번째 가우시안에 도달하는 빛의 투과율
  - 앞의 모든 가우시안(1번부터 i-1번까지)을 통과한 빛의 양
- **$\prod_{j=1}^{i-1}(1-\alpha_j)$**: 누적 투과율 계산
  - 예: 1번과 2번 가우시안이 각각 투명도 0.5라면, 3번째는 0.5 × 0.5 = 0.25만 투과

**직관 예시**:
색유리 3장을 겹쳐서 본다고 생각하세요:
- 1번 유리: 빨간색, 투명도 0.7 (30% 투과)
- 2번 유리: 파란색, 투명도 0.8 (20% 투과)  
- 3번 유리: 녹색, 투명도 1.0 (100% 투과)

최종 색상 = 빨강(0.7) + 파랑(0.8 × 0.3) + 녹색(1.0 × 0.3 × 0.2)

**렌더링 최적화**
1. **타일 기반 접근**: 이미지를 16×16 픽셀 타일로 분할
   - GPU 캐시 효율 향상
   - 스레드 병렬 처리 최적화
   
2. **깊이 정렬**: 각 타일 내에서 가우시안을 깊이 순서로 정렬
   - 카메라에서 가까운 것부터 먼 것 순서로 정렬
   
3. **조기 종료(Early Termination)**: 투명도 누적이 임계값에 도달하면 계산 중단
   - 누적된 투명도가 0.9999에 도달하면 (남은 투명도 < 0.0001) 뒤의 가우시안 무시
   - 필요 없는 계산 제거로 속도 향상
   
4. **CUDA 병렬화**: GPU에서 효율적인 병렬 처리

## 핵심 개념 정리

### 기술적 개념

#### 1. 가우시안 표현의 장점

- **명시적 기하학**: 가우시안의 위치와 크기가 명확하게 정의됨
- **미분 가능성**: 모든 파라미터에 대해 미분 가능 (SGD 최적화 가능)
- **유연성**: 이방성 공분산으로 임의의 형태 표현 가능
- **효율성**: 빈 공간을 자동으로 무시

#### 2. 공분산 행렬 표현

공분산 행렬은 다음과 같이 인수분해됩니다:

$$\Sigma = \mathbf{R} \mathbf{S} \mathbf{S}^T \mathbf{R}^T$$

**수식 설명**
이 수식은 3D 가우시안의 모양을 회전과 확대/축소의 조합으로 표현합니다:
- **$\Sigma$**: 공분산 행렬 - 전체 모양을 정의
- **$\mathbf{R}$**: 회전 행렬 - 가우시안의 방향을 결정
  - 예: (1,0,0) 방향으로 늘어난 모양
  - 쿼터니언(quaternion)으로 표현하여 계산 안정성 향상
- **$\mathbf{S}$**: 스케일 행렬 (대각 행렬) - 각 축 방향의 크기 결정
  - 예: x축 길이 2, y축 길이 1, z축 길이 1.5
  - 더 정확히는 $\mathbf{S} = \text{diag}(\sigma_x, \sigma_y, \sigma_z)$

**직관**: 공 모양의 가우시안을 회전시키고 확대/축소하여 원하는 형태를 만드는 과정입니다.

#### 3. 스플래팅(Splatting) - 3D를 2D로 투영

2D 이미지 평면에 3D 가우시안을 투영하는 과정:

$$\Sigma' = \mathbf{J} \Sigma \mathbf{J}^T$$

**수식 설명**
이 수식은 3D 가우시안이 카메라를 통해 2D 이미지에 어떻게 나타나는지 계산합니다:
- **$\Sigma'$**: 투영된 2D 가우시안의 공분산
- **$\mathbf{J}$**: 야코비안 행렬 - 3D의 작은 변화가 2D에서 얼마나 변하는지 나타냄
  - 예: 카메라에 가까운 객체는 작은 이동이 2D에서 큰 변화를 일으킴
- **$\Sigma$**: 원래 3D 가우시안의 공분산

**직관**: 카메라가 3D 장면을 2D 사진으로 찍을 때, 3D 가우시안의 모양이 어떻게 변하는지 수학적으로 계산하는 것입니다.

#### 4. 밀도 제어 알고리즘

```
훈련 반복마다:
1. 기울기(gradient)가 높은 곳: 가우시안 분할 (split)
   - 너무 큰 가우시안을 작은 것들로 나눔
   - 더 정확한 표현 가능

2. 기울기가 낮은 곳 & 투명도 < 임계값: 가우시안 제거 (prune)
   - 덜 중요한 가우시안 삭제
   - 메모리 효율성 향상

3. 투명도가 매우 높은 경우 & 크기가 큼: 투명도 리셋
   - 거의 불투명한 큰 가우시안이 너무 오래 유지되지 않도록 조정

4. 투명도가 낮은 모든 가우시안: 최종 제거
```

### 렌더링 관련 개념

#### 1. 알파 블렌딩 (Alpha Blending) - 투명도를 고려한 합성

깊이 순서로 정렬된 가우시안들을 조합:

$$C = \sum_{i=1}^{n} T_i c_i \alpha_i, \quad T_i = \prod_{j=1}^{i-1}(1-\alpha_j)$$

**이미 위에서 설명한 내용 참고**

#### 2. 타일 기반 최적화

- 16×16 픽셀 타일 단위로 처리하는 이유:
  - CPU 캐시 지역성 향상 (한 번에 많은 데이터 읽음)
  - GPU 스레드 블록 최적화 (한 번에 256개 스레드 처리)
  - 메모리 접근 패턴 규칙화
  
- 타일당 평균 100개 미만의 가우시안만 영향
  - 대부분의 타일은 소수의 가우시안으로만 렌더링됨

#### 3. 조기 종료 (Early Termination)

누적 알파(accumulated alpha)가 0.9999에 도달하면 계산 종료:

$$T_{remaining} = \prod_{j=1}^{i}(1-\alpha_j) < 0.0001$$

**수식 설명**
남은 투명도(빛의 투과율)가 매우 작으면, 더 이상 뒤의 가우시안이 보이지 않으므로 계산 중단:
- **$T_{remaining}$**: 누적 투명도 이후 남은 투명도
- **< 0.0001**: 더 이상 화면에 유의미한 영향을 주지 않음
- 99.99% 이상이 불투명해지면 중단

**효과**: 평균적으로 불필요한 계산의 30~40% 절감

## 구현 세부사항

### 손실 함수의 선택

$$\mathcal{L}_{total} = (1-\lambda) \cdot L_1(\hat{C}, C) + \lambda \cdot L_{SSIM}(\hat{C}, C)$$

**각 항의 의미**
- **$(1-\lambda) \cdot L_1$** (보통 80%):
  - 각 픽셀의 절대 오차를 직접 계산
  - 색상 정확도에 민감
  
- **$\lambda \cdot L_{SSIM}$** (보통 20%):
  - 이미지의 구조적 유사성 측정
  - 선명도, 대비, 구조 보존
  
- **$\lambda = 0.2$**: 실험적으로 최적이 증명됨

### 최적화기 설정

- **옵티마이저**: Adam (적응형 학습률 사용)
- **학습률 설정**:
  - Position (위치): $10^{-3}$에서 시작하여 지수적으로 감소
  - Other (회전, 스케일 등): $10^{-2}$에서 시작
  - 감소 방식: 훈련 진행에 따라 자동 감소
  
- **배치 처리**:
  - 전체 이미지 또는 랜덤 샘플 사용
  - 한 번에 여러 이미지의 픽셀들을 섞어서 처리
  
- **반복 횟수**: 30,000~50,000 반복 (대부분 7~51분 소요)

- **건축 시각화**: 신축 건물의 사실적 미리보기

- 불과 수십 장의 이미지로부터 고품질 3D 장면 재구성

- 모바일 디바이스에서 경량 모델 배포 가능

### 3. 학습 시간의 획기적 단축

- 48시간 → 51분 (약 57배 단축)
- 병렬 처리와 효율적 알고리즘의 중요성 입증
- GPU 활용도 극대화의 중요성

### 4. 기하학과 신경 표현의 융합

- 순수 신경망 방식의 한계 극복
- 기하학적 직관과 심층 학습의 결합
- 명시적 표현의 해석 가능성 향상

### 5. 향후 연구 방향

**제한사항**
- 동적 장면 처리 미흡 (정적 장면 가정)
- 매우 큰 실외 장면의 메모리 효율성 문제
- 투명성 있는 물체 표현의 어려움

**확장 가능성**
- 시간 차원 추가로 동적 장면 지원
- 계층적 가우시안 표현으로 대규모 장면 지원
- 조건부 생성으로 스타일 제어 가능성

## 결론 및 시사점

**핵심 기여**

이 논문은 단순하면서도 혁신적인 아이디어를 통해 3D 장면 렌더링의 패러다임을 전환했습니다:

1. **3D 가우시안 표현**: 최적화와 렌더링의 효율성이 뛰어난 새로운 표현
2. **효율적인 렌더링**: 타일 기반 스플래팅으로 GPU 최대 활용
3. **실시간 가능성**: 실시간 고품질 렌더링 처음 달성

**학문적 의미**

- 신경 표현과 기하학적 표현의 경계 무너뜨림
- 렌더링 알고리즘의 GPU 친화성 재조명
- 단순한 표현이 복잡한 신경망보다 우수할 수 있음 증명

**산업적 영향**

- 메타(Meta), 구글, 애플 등 대형 기술 회사들이 후속 연구 진행
- 3D 디지털화의 실시간 처리 시대 개막
- 메타버스, XR 기술의 현실화 가속

**종합 평가**

2023년 SIGGRAPH 최우수 논문 중 하나로 평가되며, 3D 그래픽스 분야에 깊은 영향을 미치고 있습니다. 단순하면서도 강력한 아이디어, 철저한 실험, 그리고 즉각적인 실용성이 결합된 모범적 연구입니다.
