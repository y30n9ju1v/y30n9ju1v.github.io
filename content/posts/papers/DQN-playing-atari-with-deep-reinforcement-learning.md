## 개요

- **저자**: Volodymyr Mnih, Koray Kavukcuoglu, David Silver, Alex Graves, Ioannis Antonoglou, Daan Wierstra, Martin Riedmiller (DeepMind Technologies)
- **발행년도**: 2013 (arXiv:1312.5602)
- **주요 내용**: 원시 픽셀 이미지만을 입력으로 받아 강화학습으로 Atari 2600 게임을 스스로 학습하는 최초의 딥러닝 모델 **DQN(Deep Q-Network)** 제안. Experience Replay와 CNN을 결합한 Q-learning으로 7개 게임 중 6개에서 기존 방법 전체를 능가하고, 3개 게임에서 인간 전문가를 초월.

## 한계 극복

이 논문은 딥러닝과 강화학습을 결합하려는 기존 연구들이 직면한 세 가지 핵심 한계를 극복하기 위해 작성되었습니다.

- **기존 한계 1 — 수동 피처 의존**: 이전 RL 시스템들은 hand-crafted 피처와 선형 함수 근사에 의존했으며, raw 감각 입력(픽셀 등)에서 직접 학습하는 딥러닝 모델은 존재하지 않았습니다.
- **기존 한계 2 — 비선형 Q-learning의 발산**: Q-learning과 비선형 함수 근사기(신경망)를 결합하면 Q값이 발산하는 것으로 알려져 있었고, 이로 인해 대부분의 연구가 수렴 보장이 있는 선형 근사에 머물렀습니다.
- **기존 한계 3 — 데이터 상관관계와 비정상 분포**: RL 환경에서 연속된 상태들은 강하게 상관되어 있고, 에이전트가 학습할수록 데이터 분포가 변하는 문제가 있어 독립·고정 분포를 가정하는 딥러닝 적용이 어려웠습니다.
- **이 논문의 접근 방식**: CNN 기반 Q-network와 **Experience Replay**를 결합하여, 과거 경험을 무작위 샘플링함으로써 상관관계와 분포 문제를 해소하고, raw 픽셀에서 end-to-end로 직접 제어 정책을 학습하는 데 성공했습니다.

## 1. Introduction: 딥러닝과 강화학습의 결합 과제

**요약**

시각·청각처럼 고차원 감각 입력에서 직접 제어 정책을 학습하는 것은 강화학습의 오랜 난제였습니다. 기존 성공적인 RL 시스템들은 손으로 만든(hand-crafted) 피처에 의존했고, 딥러닝의 등장으로 raw 데이터에서 고수준 피처를 자동으로 추출할 수 있게 되었습니다. 그러나 딥러닝을 RL에 적용하는 데는 세 가지 근본적 장애물이 있습니다:

1. **희소·지연 보상**: RL의 보상은 드물고, 노이즈가 많으며, 행동 후 수천 타임스텝이 지나서야 도달할 수 있음
2. **데이터 상관관계**: RL의 연속적 상태들은 강하게 상관되어 있어, 독립적 샘플을 가정하는 딥러닝과 충돌
3. **비정상 분포**: 에이전트가 새로운 행동을 학습할수록 데이터 분포가 바뀌어 학습 불안정

이 논문은 **Experience Replay**로 이 문제들을 해결하고, CNN 기반의 Q-network로 Atari 게임을 raw 픽셀에서 직접 학습합니다.

**핵심 개념**

- **강화학습(Reinforcement Learning)**: 에이전트가 환경과 상호작용하며 보상을 최대화하도록 행동 정책을 학습하는 방법
- **희소 보상(Sparse Reward)**: 게임에서 점수가 오르는 순간만 보상이 주어지고 나머지 타임스텝은 0
- **Experience Replay**: 과거 경험을 메모리에 저장해두고 랜덤하게 샘플링하여 학습 → 상관관계와 분포 문제 해소

## 2. Background: MDP, Q-learning, 벨만 방정식

**요약**

에이전트는 매 타임스텝마다 행동을 선택하고 보상과 다음 화면을 관찰합니다. 게임의 내부 상태는 알 수 없으므로 (부분 관측 문제), 과거 프레임의 시퀀스 $s_t = x_1, a_1, x_2, \ldots, a_{t-1}, x_t$를 상태로 사용합니다.

**미래 할인 보상:**

$$R_t = \sum_{t'=t}^{T} \gamma^{t'-t} r_{t'}$$

**수식 설명**

- **$R_t$**: 타임스텝 $t$부터 에피소드 끝까지 받을 미래 보상의 합
- **$r_{t'}$**: 타임스텝 $t'$에서 받는 보상
- **$\gamma \in [0, 1]$**: 할인 인수(discount factor). 미래 보상을 현재보다 덜 중요하게 취급
  - $\gamma = 0$이면 현재 보상만 고려, $\gamma = 1$이면 먼 미래까지 동등하게 고려
  - 직관: 지금 당장 받는 1점이 100타임스텝 후의 1점보다 더 가치 있음
- **$T$**: 게임이 종료되는 타임스텝

**최적 행동-가치 함수(벨만 방정식):**

$$Q^*(s, a) = \mathbb{E}_{s' \sim \mathcal{E}}\left[r + \gamma \max_{a'} Q^*(s', a') \,\Big|\, s, a\right]$$

**수식 설명**

이것이 DQN의 핵심 이론인 **벨만 방정식(Bellman Equation)**입니다:

- **$Q^*(s, a)$**: 상태 $s$에서 행동 $a$를 취할 때 얻을 수 있는 **최대 기대 보상** (최적 Q값)
- **$s'$**: 행동 $a$를 취한 후 전이되는 다음 상태
- **$r$**: 즉각적으로 받는 보상
- **$\gamma \max_{a'} Q^*(s', a')$**: 다음 상태에서 최선의 행동을 했을 때 기대되는 미래 보상 (할인됨)
- **직관**: "지금 받는 보상 + 다음 상태에서 최선의 행동으로 얻을 보상" = 현재 행동의 총 가치

**Q-network 손실 함수:**

$$L_i(\theta_i) = \mathbb{E}_{s,a \sim \rho(\cdot)}\left[\left(y_i - Q(s, a; \theta_i)\right)^2\right]$$

$$\text{where } y_i = \mathbb{E}_{s' \sim \mathcal{E}}\left[r + \gamma \max_{a'} Q(s', a'; \theta_{i-1}) \,\Big|\, s, a\right]$$

**수식 설명**

Q-network를 학습시키기 위한 손실 함수입니다:

- **$\theta_i$**: 현재 이터레이션 $i$의 네트워크 가중치
- **$y_i$**: 타깃값(target). "이 상태-행동 쌍의 Q값은 이 정도여야 한다"는 목표
- **$Q(s, a; \theta_i)$**: 현재 네트워크가 예측하는 Q값
- **$(y_i - Q)^2$**: 예측값과 목표값의 차이를 최소화 (지도학습의 MSE와 유사)
- **핵심 차이**: 타깃 $y_i$가 이전 이터레이션의 가중치 $\theta_{i-1}$로 계산됨 → 타깃이 고정되지 않고 점진적으로 변함

**Q-network 그래디언트:**

$$\nabla_{\theta_i} L_i(\theta_i) = \mathbb{E}\left[\left(r + \gamma \max_{a'} Q(s', a'; \theta_{i-1}) - Q(s, a; \theta_i)\right) \nabla_{\theta_i} Q(s, a; \theta_i)\right]$$

**수식 설명**

손실을 가중치로 미분한 그래디언트입니다:

- **괄호 안의 값**: TD 오차(Temporal Difference Error) = 타깃과 예측의 차이
- **$\nabla_{\theta_i} Q$**: Q값을 가중치로 미분한 값 (역전파)
- **직관**: TD 오차가 클수록 더 크게 가중치를 업데이트

**핵심 개념**

- **MDP(Markov Decision Process)**: 상태, 행동, 보상, 전이 확률로 정의되는 의사결정 수학적 모델
- **Q-값(Action-Value)**: 특정 상태에서 특정 행동을 취했을 때 기대되는 총 미래 보상
- **Value Iteration**: 벨만 방정식을 반복 적용하면 $Q_i \to Q^*$로 수렴
- **Model-free**: 환경의 전이 모델을 직접 추정하지 않고 샘플로 학습
- **Off-policy**: 탐험 정책(ε-greedy)과 학습 정책(greedy)이 다름

## 3. Related Work

**요약**

- **TD-Gammon**: 강화학습으로 백가몬을 초인간 수준으로 학습한 선구적 사례. MLP + TD-learning 사용. 하지만 체스, 바둑에는 적용 실패 → "TD-Gammon은 특수 케이스"라는 인식 확산
- **비선형 Q-learning의 발산 문제**: 비선형 함수 근사기(신경망)와 Q-learning을 결합하면 발산하는 것으로 알려짐. 이후 연구들은 선형 함수 근사에 집중
- **NFQ(Neural Fitted Q-learning)**: Q-network 손실을 배치로 최적화. 하지만 배치 비용이 데이터셋 크기에 비례 → 확장성 한계
- **ALE(Arcade Learning Environment)**: Atari 2600을 RL 벤치마크로 도입. 선형 함수 근사 + 수동 피처에서 시작

DQN은 위 선행 연구들의 문제를 Experience Replay와 함께 **raw 픽셀 → end-to-end 학습**으로 해결합니다.

## 4. Deep Reinforcement Learning: DQN과 Experience Replay

**요약**

DQN의 핵심은 두 가지입니다: **CNN 기반 Q-network**와 **Experience Replay**.

### Experience Replay

각 타임스텝의 경험 $e_t = (s_t, a_t, r_t, s_{t+1})$을 리플레이 메모리 $\mathcal{D} = \{e_1, \ldots, e_N\}$에 저장합니다. 학습 시에는 $\mathcal{D}$에서 무작위로 미니배치를 샘플링합니다.

**Experience Replay의 세 가지 이점**:
1. **데이터 효율성**: 하나의 경험이 여러 번 학습에 사용됨
2. **상관관계 제거**: 연속된 경험 대신 랜덤 샘플링 → 업데이트 분산 감소
3. **분포 안정화**: 현재 정책에 의존하지 않고 과거 다양한 경험의 평균으로 학습 → 발산·진동 방지

### DQN 알고리즘 (Algorithm 1)

```
리플레이 메모리 D를 용량 N으로 초기화
Q-network를 랜덤 가중치로 초기화
for 에피소드 = 1 to M:
    초기 상태 s₁ 설정, 전처리 φ₁ = φ(s₁)
    for t = 1 to T:
        확률 ε로 랜덤 행동 aₜ 선택 (탐험)
        그 외에는 aₜ = argmax_a Q(φ(sₜ), a; θ) 선택 (활용)
        에뮬레이터에서 행동 실행 → 보상 rₜ, 다음 프레임 xₜ₊₁ 관찰
        전이 (φₜ, aₜ, rₜ, φₜ₊₁)를 D에 저장
        D에서 랜덤 미니배치 샘플링
        타깃 yⱼ 계산 후 그래디언트 하강
```

**타깃 계산:**

$$y_j = \begin{cases} r_j & \text{종료 상태인 경우} \\ r_j + \gamma \max_{a'} Q(\phi_{j+1}, a'; \theta) & \text{비종료 상태인 경우} \end{cases}$$

**수식 설명**

- **종료 상태**: 게임이 끝났으므로 미래 보상 없음 → 즉각 보상 $r_j$만이 타깃
- **비종료 상태**: 즉각 보상 + 다음 상태에서 최선의 행동으로 얻을 할인된 미래 보상
- **직관**: "이번 판에서 죽으면 $r_j$점이 전부, 살아있으면 $r_j$점 + 앞으로 더 받을 점수"

### 4.1 전처리 및 모델 아키텍처

**입력 전처리**: 210×160 RGB → 그레이스케일 → 110×84 다운샘플링 → 84×84 크롭. 최근 4프레임을 쌓아 **84×84×4** 텐서를 Q-network 입력으로 사용.

**DQN 구조**:

| 레이어 | 설명 |
|--------|------|
| 입력 | 84 × 84 × 4 (전처리된 4프레임) |
| Conv1 | 16개 8×8 필터, stride 4, ReLU |
| Conv2 | 32개 4×4 필터, stride 2, ReLU |
| FC | 256 ReLU 유닛 |
| 출력 | 각 행동에 대한 Q값 (4~18개) |

**설계 핵심**: 입력이 상태만이고 출력이 모든 행동의 Q값 → 한 번의 순전파로 모든 행동의 Q값 계산 가능 (이전 방식은 행동마다 별도 순전파 필요)

**핵심 개념**

- **ε-greedy 정책**: 확률 $\epsilon$으로 랜덤 탐험, 확률 $1-\epsilon$으로 최적 행동 선택. 훈련 중 $\epsilon$을 1.0에서 0.1로 선형 감소
- **프레임 스킵(Frame Skip)**: 매 $k$번째 프레임만 처리. 계산 비용 절감 + 더 많은 게임 경험
- **보상 클리핑(Reward Clipping)**: 양수 보상 → +1, 음수 → -1, 0 유지. 게임 간 보상 스케일 차이 해소

## 5. Experiments

**요약**

7개 게임(Beam Rider, Breakout, Enduro, Pong, Q*bert, Seaquest, Space Invaders)에서 **동일한 아키텍처와 하이퍼파라미터**로 평가.

**학습 설정**:
- 옵티마이저: RMSProp, 미니배치 크기 32
- ε-greedy: 처음 100만 프레임 동안 1.0→0.1 선형 감소, 이후 0.1 고정
- 총 학습: 1000만 프레임
- 리플레이 메모리: 최근 100만 프레임

### 5.1 학습 안정성

평균 총 보상은 매우 잡음이 많아 학습 진행을 추적하기 어렵습니다. 대신 **평균 최대 Q값**이 훨씬 매끄럽게 증가하여 안정적인 학습 지표로 활용됩니다. 어떤 게임에서도 Q값 발산은 관찰되지 않았습니다.

### 5.2 가치 함수 시각화

Seaquest 게임의 30프레임 세그먼트에서:
- **A**: 화면 왼쪽에 적이 등장 → Q값 급등 (위협 인식)
- **B**: 어뢰가 적에게 명중 직전 → Q값 최고점 (보상 임박)
- **C**: 적 소멸 → Q값 원래 수준으로 하락

### 5.3 성능 비교

| 방법 | Beam Rider | Breakout | Enduro | Pong | Q*bert | Seaquest | Space Invaders |
|------|-----------|---------|--------|------|--------|---------|----------------|
| Random | 354 | 1.2 | 0 | -20.4 | 157 | 110 | 179 |
| Sarsa | 996 | 5.2 | 129 | -19 | 614 | 665 | 271 |
| Contingency | 1743 | 6 | 159 | -17 | 960 | 723 | 268 |
| **DQN** | **4092** | **168** | **470** | **20** | **1952** | **1705** | **581** |
| Human | 7456 | 31 | 368 | -3 | 18900 | 28010 | 3690 |

- **7개 게임 중 6개**에서 기존 모든 방법 능가
- **Breakout, Enduro, Pong**: 인간 전문가 초월
- **Q*bert, Seaquest, Space Invaders**: 인간에 못 미침 → 장기 전략 필요한 게임에서 한계

| **벨만 방정식** | 현재 Q값 = 즉각 보상 + 할인된 미래 최대 Q값 |
| **Experience Replay** | 과거 경험을 메모리에 저장하고 랜덤 샘플링하여 학습 → 상관관계·분포 문제 해소 |
| **ε-greedy** | 확률 ε로 무작위 탐험, 1-ε으로 최적 행동 선택하는 탐험-활용 균형 전략 |
| **할인 인수 γ** | 미래 보상의 현재 가치 비율. γ가 클수록 먼 미래도 중요하게 여김 |
| **TD 오차** | 타깃(벨만 방정식) - 예측(현재 Q값)의 차이. 이를 최소화하도록 학습 |
| **프레임 스킵** | 매 k번째 프레임만 처리하여 계산 효율 향상 |
| **보상 클리핑** | 보상을 {-1, 0, 1}로 정규화하여 게임 간 스케일 차이 해소 |
| **ALE** | Arcade Learning Environment. Atari 게임을 RL 연구 플랫폼으로 제공 |

## 결론 및 시사점

DQN은 **"raw 픽셀 → 직접 제어"** 라는 end-to-end 딥 강화학습의 실현 가능성을 최초로 입증한 기념비적 논문입니다.

**핵심 기여**:
- 딥러닝과 Q-learning을 성공적으로 결합한 최초 사례
- Experience Replay로 상관관계와 비정상 분포 문제 해결
- 단일 아키텍처·하이퍼파라미터로 다양한 게임에서 범용 학습 시연

**역사적 의의**:
- 이 논문은 DeepMind가 Google에 4억 달러에 인수되는 계기가 된 연구
- 이후 DQN Nature 버전(2015)에서 Target Network 도입으로 더욱 안정화
- Double DQN, Dueling DQN, Prioritized Replay 등 수많은 후속 연구의 출발점

**한계**:
- **균등 Experience Replay**: 중요한 전환(transition)과 평범한 전환을 동일하게 취급 → Prioritized Replay로 개선
- **장기 전략의 한계**: Q*bert, Seaquest처럼 수천 타임스텝 이상의 장기 계획이 필요한 게임에서 인간에 크게 못 미침
- **보상 클리핑**: 보상의 크기 정보 손실 → 미묘한 보상 차이 구별 불가
- **이론적 수렴 보장 없음**: 비선형 함수 근사 + 오프폴리시 학습이지만 실험적으로 안정성 확인
