## 1. 소개 및 배경

**요약**

자율주행 자동차는 복잡한 동적 장면에서 고품질의 3D 재구성이 필요합니다. 기존 방법들은 다음과 같은 문제점을 가집니다:

- **NeRF 기반 방법**: 계산 비용이 크고, 정적 배경에서는 잘 작동하지만 동적 객체가 많은 장면에서 어려움
- **3D Gaussian Splatting (3D-GS)**: 빠른 렌더링이 가능하지만, 다중 카메라 멀티뷰에서 동적 객체와 배경을 동시에 처리하기 어려움
- **멀티 오브젝트 문제**: 고속으로 움직이는 자동차와 보행자 같은 여러 동적 객체를 정확히 표현하기 복잡

이 논문은 **Composite Gaussian Splatting** 프레임워크를 제안하여 정적 배경과 동적 객체를 분리하여 처리합니다.

**핵심 개념**

- **Incremental Static 3D Gaussians (IS3G)**: 자동차의 이동에 따라 정적 배경을 점진적으로 구성하는 방식
- **Composite Dynamic Gaussian Graph**: 여러 동적 객체를 그래프 구조로 표현하고 시간에 따라 통합
- **LiDAR Prior**: 기하 정보를 정확하게 초기화하기 위해 LiDAR 포인트 클라우드를 활용

## 2. 관련 연구

**요약**

**NeRF for Bounded Scenes**
- Neural Radiance Fields (NeRF)는 다층 퍼셉트론(MLP)과 미분 가능한 볼륨 렌더링으로 3D 장면을 재구성
- 장점: 고품질의 새로운 뷰 합성
- 한계: 계산 집약적이고 경계가 있는 장면에만 효과적

**NeRF for Unbounded Scenes**
- 대규모 실외 장면을 다루기 위해 여러 개선 방법 제안
- 계산 비용 감소와 품질 개선을 위해 LiDAR를 보조 감독(auxiliary supervision)으로 활용

**3D Gaussian Splatting (3D-GS)**
- 정적 장면에서 탁월한 성능 달성 (novel view synthesis와 렌더링 속도)
- 한계: 동적 객체와 배경이 섞인 장면에서 의도하지 않은 신공동변수 공유로 인한 문제 발생

**동적 3D 가우시안 표현**
- 일부 연구에서 3D-GS를 동적 객체로 확장
- Deformable 3D Gaussians와 움직임 필드를 사용하는 방식 제안
- 한계: 다중 객체, 빠른 움직임, 멀티카메라 설정에서 여전히 어려움

**핵심 개념**

- **장면 그래프 (Scene Graph)**: 정적 배경과 동적 객체를 계층적으로 표현
- **LiDAR 감독**: 기하학적 정확성을 보장하기 위한 깊이 정보 제약

## 3. 방법론 - Composite Gaussian Splatting

### 3.1 Incremental Static 3D Gaussians

**요약**

자율주행 장면에서 가장 큰 도전은 자동차가 이동하면서 배경이 연속적으로 변한다는 것입니다. 이를 해결하기 위해 정적 배경을 **시간에 따라 누적적으로 구성**합니다.

**프로세스**

1. **정적 장면 분할**: LiDAR 깊이 정보를 이용해 장면을 N개의 깊이 범위 "빈(bin)"으로 나눔

$$P_{h+1}(G_s) = P_h(G_s) \cup \{(x_{h+1}, y_{h+1}, z_{h+1})\}  \quad \text{(수식 2)}$$

**수식 설명**:
- **$P_h(G_s)$**: h번째 시간 단계에서 정적 가우시안의 3D 중심 좌표 집합
- **$\{(x_{h+1}, y_{h+1}, z_{h+1})\}$**: h+1 시간에 새로 추가되는 가우시안의 좌표
- **$\cup$**: 합집합 연산으로, 이전의 모든 가우시안에 새 가우시안을 추가한다는 의미
- **직관**: 자동차가 새로운 영역으로 이동할 때마다 해당 영역의 가우시안을 누적적으로 추가

2. **초기 가우시안 구성**: 첫 번째 빈에서 LiDAR 프라이어를 사용하여 가우시안 모델 초기화

$$p_{lh}(l \mid \mu, \Sigma) = e^{-\frac{1}{2}(l-\mu)^T \Sigma^{-1}(l-\mu)}  \quad \text{(수식 1)}$$

**수식 설명**:
- **$l$**: LiDAR 포인트의 3D 위치 (우리가 관찰한 깊이 정보)
- **$\mu$**: 가우시안의 중심 위치 (배우고 싶은 값)
- **$\Sigma$**: 가우시안의 공분산 행렬 (불확실성을 나타냄)
- **$e^{-\frac{1}{2}(l-\mu)^T \Sigma^{-1}(l-\mu)}$**: 가우시안 확률 분포 함수
- **직관**: LiDAR 포인트를 중심으로 가우시안을 초기화하여 정확한 기하학적 위치에서 시작

3. **가우시안 누적**: 각 빈의 가우시안을 개별적으로 최적화하여 배경 일관성 유지

**핵심 개념**

- **Perspective Correction**: 인접한 카메라 뷰에서 일관된 가우시안 위치를 보장하기 위해 가중 평균 사용

$$\bar{C} = ς(G_s) \sum \omega(C(G_s)) [R, T]$$

**수식 설명**:
- **$\bar{C}$**: 최종 최적화된 픽셀 색상
- **$ς(G_s)$**: 차등 분할 (differential splatting) 함수
- **$\sum \omega(C(G_s))[R, T]$**: 여러 뷰에서 가중 평균한 색상
- **$[R, T]$**: 뷰 매트릭스 (회전 R과 평행이동 T)
- **직관**: 여러 카메라에서 본 색상을 적절하게 혼합하여 일관된 결과 도출

### 3.2 Composite Dynamic Gaussian Graph

**요약**

동적 객체(자동차, 보행자 등)는 배경과 다르게 처리됩니다. 각 객체를 개별 가우시안으로 표현하고, 이들을 **그래프 구조**로 연결하여 시간에 따른 변화를 추적합니다.

**그래프 구조**

$$G_{comp} = \sum \langle O, G_d, M, P, A, T \rangle + G_s  \quad \text{(수식 8)}$$

**수식 설명**:
- **$O$**: Object ID - 각 동적 객체의 고유 식별자
- **$G_d$**: 해당 객체의 동적 가우시안들
- **$M$**: Transform Matrix - 객체의 회전과 이동 정보
- **$P$**: 바운딩 박스의 중심 좌표 (객체의 위치)
- **$A$**: 나타나는 시간 범위 (appearance timestamps)
- **$T$**: 시간 단계 인덱스
- **$G_s$**: 정적 배경 가우시안들
- **직관**: 각 동적 객체는 독립적인 가우시안 집합을 가지면서도, 그래프로 연결되어 상호작용 관계를 표현

**객체 추적 및 분해**

1. **객체 분할**: 3D 바운딩 박스를 통해 동적 객체를 자동으로 식별
2. **Grounded SAM Models**: 객체 분할의 정확성을 위해 사전 학습된 모델 활용
3. **시간 추적**: 각 객체의 나타나는 시간과 사라지는 시간 기록

### 3.3 Global Rendering via Gaussian Splatting

**요약**

정적 배경과 동적 객체 모두를 전역적으로 렌더링하기 위해 차등 3D 가우시안 분할기를 사용합니다.

**공분산 투영**

$$\tilde{\Sigma} = JE\Sigma E^T J^T  \quad \text{(수식 10)}$$

**수식 설명**:
- **$\tilde{\Sigma}$**: 2D 이미지 평면에 투영된 가우시안의 공분산 (화면에서의 "흐릿함")
- **$J$**: Jacobian 행렬 (원근 투영의 미분)
- **$E$**: 고유벡터 행렬 (회전 성분)
- **$\Sigma$**: 원래의 3D 공분산 행렬 (3D 공간에서의 "흐릿함")
- **$E^T$**: 고유벡터 행렬의 전치
- **$J^T$**: Jacobian의 전치
- **직관**: 3D 공간의 가우시안을 카메라 관점에서 2D 이미지로 투영할 때, 그 모양과 크기가 어떻게 변하는지를 계산

**투명도 합성 (Alpha Compositing)**

$$C = \sum_{i \in N} c_i \alpha_i \prod_{j=1}^{i-1}(1 - \alpha_j)  \quad \text{(수식 기존)}$$

**수식 설명**:
- **$C$**: 최종 출력 색상 (화면에 보이는 색)
- **$c_i$**: i번째 가우시안의 색상
- **$\alpha_i$**: i번째 가우시안의 투명도 (0 = 완전 투명, 1 = 완전 불투명)
- **$\sum_{i \in N}$**: 모든 가우시안을 앞에서 뒤로 순서대로 합함
- **$\prod_{j=1}^{i-1}(1 - \alpha_j)$**: i번째 가우시안 앞의 모든 가우시안을 통과한 빛의 투과율
  - 예: 1번째 가우시안이 투명도 0.5면, 뒤의 가우시안은 50%만 보임
- **직관**: 여러 반투명 층을 쌓을 때, 각 층의 색상과 투명도를 순서대로 고려하여 최종 색상을 계산

## 4. LiDAR 프라이어를 통한 기하 정보 개선

**요약**

LiDAR는 정확한 3D 기하 정보를 제공합니다. 이를 활용하여 가우시안의 초기화와 최적화 과정에서 기하학적 정확성을 보장합니다.

**멀티뷰 등록 (Multi-view Registration)**

**프로세스**

1. **LiDAR 포인트 클라우드 통합**: 여러 프레임의 LiDAR 스캔을 병합하여 완전한 포인트 클라우드 생성

2. **카메라-LiDAR 정렬**: Colmap과 이미지 특징(feature) 추출을 통해 카메라 좌표계와 LiDAR 포인트를 정렬

$$x_p^i = K[R_i^t + T_i^t] x_p  \quad \text{(수식 9)}$$

**수식 설명**:
- **$x_p^i$**: i번째 카메라 이미지 평면의 2D 픽셀 좌표
- **$K$**: 카메라 내재 행렬 (카메라의 초점거리, 주점 등 고정 파라미터)
- **$R_i^t$**: i번째 카메라의 회전 행렬
- **$T_i^t$**: i번째 카메라의 평행이동 벡터
- **$x_p$**: 3D 월드 좌표계의 LiDAR 포인트
- **직관**: 3D 공간의 점이 i번째 카메라의 이미지 평면에 어떤 위치(픽셀 좌표)로 투영되는지를 계산

3. **적응적 초기화**: 다운샘플된 LiDAR 포인트 클라우드를 사용하여 계산 비용 절감

**손실 함수 - LiDAR 감독**

$$L_{LiDAR}(\delta) = \frac{1}{S} \sum ||P(G_{comp}) - L_*||^2  \quad \text{(수식 13)}$$

**수식 설명**:
- **$L_{LiDAR}(\delta)$**: LiDAR 손실값 (작을수록 좋음)
- **$P(G_{comp})$**: 가우시안들로부터 투영된 3D 위치
- **$L_*$**: LiDAR로부터 얻은 실제 3D 위치
- **$||...||^2$**: 유클리드 거리의 제곱 (두 위치 사이의 거리)
- **$\frac{1}{S}$**: 평균을 구하기 위해 전체 포인트 수로 나눔
- **직관**: 가우시안으로 예측한 위치가 실제 LiDAR 포인트와 얼마나 다른지를 측정

## 5. 손실 함수 및 학습

**요약**

세 가지 손실 함수를 결합하여 높은 품질의 재구성을 달성합니다.

**Tile Structural Similarity (SSIM)**

$$L_{SSIM}(\delta) = 1 - \frac{1}{Z} \sum SSIM(\Psi(C), \Psi(C))  \quad \text{(수식 11)}$$

**수식 설명**:
- **$L_{SSIM}$**: SSIM 손실값 (1에 가까울수록 나쁨, 0에 가까울수록 좋음)
- **$\Psi(C)$**: 렌더링된 이미지를 타일(tile)로 분할
- **$\Psi(C')$**: 실제 이미지를 같은 방식으로 분할
- **$SSIM(..., ...)$**: 두 이미지 타일의 구조적 유사성 (밝기, 명도, 구조 모두 고려)
- **직관**: 전체 이미지가 아닌 작은 패치 단위로 비교하여 국지적 구조 정확도 측정

**Robust Loss (이상치 제거)**

$$L_{Robust}(\delta) = \kappa(||I - I_*||_2)  \quad \text{(수식 12)}$$

**수식 설명**:
- **$L_{Robust}$**: 강건한 손실값
- **$\kappa \in [0, 1]$**: 형태 매개변수 (0이면 이상치에 민감, 1이면 둔감)
- **$||I - I_*||_2$**: 렌더링 이미지 I와 실제 이미지 I* 사이의 L2 거리
- **직관**: 극단적으로 다른 픽셀(이상치)의 영향을 줄여서, 대부분의 정상적인 데이터에 집중

## 6. 실험 및 결과

### 6.1 데이터셋

**nuScenes Dataset**
- 대규모 자율주행 데이터셋 (1000개 장면)
- 멀티센서: 6개 카메라, 1개 LiDAR, GPS 등
- 23개 객체 클래스의 3D 바운딩 박스 포함
- 6개의 도전적인 시나리오 평가

**KITTI-360 Dataset**
- 단안 카메라와 LiDAR 포함
- 대규모 도시 드라이빙 장면
- 메트릭: PSNR, SSIM, LPIPS

### 6.2 비교 결과

**nuScenes에서의 성능 비교**

| 방법 | 입력 | PSNR ↑ | SSIM ↑ | LPIPS ↓ |
|------|------|--------|--------|---------|
| NeRF | Images | 17.54 | 0.565 | 0.532 |
| 3D-GS | Images + SIM Points | 26.08 | 0.717 | 0.298 |
| EmerNeRF | Images + LiDAR | 26.75 | 0.760 | 0.311 |
| **DrivingGaussian (Ours-L)** | **Images + LiDAR** | **28.74** | **0.865** | **0.237** |

**핵심 결과**
- **PSNR**: 기존 방법 대비 8% 향상 (이미지 품질)
- **SSIM**: 기존 최고 대비 13.8% 향상 (구조적 유사성)
- **LPIPS**: 기존 방법 대비 23.8% 향상 (인지적 차이 감소)

**KITTI-360에서의 성능**

| 방법 | PSNR ↑ | SSIM ↑ |
|------|--------|--------|
| NeRF | 21.94 | 0.781 |
| 3D-GS | 23.34 | 0.846 |
| **DrivingGaussian (Ours-L)** | **25.62** | **0.868** |

### 6.3 Ablation Study (각 모듈의 효과)

**LiDAR Prior의 효과**

| 초기화 방법 | PSNR ↑ | SSIM ↑ | LPIPS ↓ |
|-----------|--------|--------|---------|
| Random | 22.18 | 0.653 | 0.424 |
| Random + SIM | 22.23 | 0.653 | 0.421 |
| LiDAR | 28.49 | 0.854 | 0.245 |
| **LiDAR + Adaptive** | **28.78** | **0.867** | **0.237** |

**해석**: LiDAR 프라이어만으로도 큰 성능 향상, 적응적 초기화로 추가 개선

**각 모듈의 기여도**

| 모듈 | PSNR ↑ | SSIM ↑ | LPIPS ↓ |
|-----|--------|--------|---------|
| w/o IS3G | 27.72 | 0.771 | 0.363 |
| w/o CDGG | 26.97 | 0.752 | 0.306 |
| w/o LiDAR Supervision | 28.05 | 0.814 | 0.271 |
| **전체 모델** | **28.74** | **0.865** | **0.237** |

**해석**:
- IS3G (정적 배경): 성능의 ~1% 영향
- CDGG (동적 객체): 성능의 ~1% 영향
- LiDAR Supervision: 성능의 ~0.7% 영향
- 각 모듈이 협력하여 최고 성능 달성

### 6.4 정성적 결과 (Qualitative Results)

**동적 재구성**
- EmerNeRF와 3D-GS는 고속 이동 객체에서 흐릿함, 왜곡 발생
- DrivingGaussian은 명확한 객체 윤곽선 유지

**코너 케이스 시뮬레이션**
- 임의 객체 삽입/제거 가능
- 멀티카메라 일관성 유지
- 실제 자율주행 시뮬레이션에 활용 가능

## 7. 핵심 개념 정리

### 주요 기술들

**Incremental Static 3D Gaussians (IS3G)**
- **개념**: 자동차의 이동에 따라 배경을 점진적으로 구성
- **장점**: 대규모 장면을 메모리 효율적으로 처리
- **관계**: NeRF의 "volume rendering"과 유사하지만, 가우시안을 사용하여 고속 렌더링 실현

**Composite Dynamic Gaussian Graph (CDGG)**
- **개념**: 동적 객체들을 그래프 구조로 연결
- **장점**: 여러 객체를 독립적으로 처리하면서도 상호작용 표현
- **관계**: 장면 그래프(Scene Graph)와 유사한 구조화된 표현

**LiDAR Prior Integration**
- **개념**: LiDAR 포인트 클라우드를 기하 정보로 활용
- **장점**: 기존 방법의 주요 약점인 기하 부정확성 해결
- **관계**: 자기교사학습(Self-supervised learning)의 원리 - 다른 센서의 정보를 감독 신호로 사용

**차등 3D 가우시안 분할 (Differentiable 3D GS)**
- **개념**: 가우시안을 이미지 평면에 투영 가능하게 만듦
- **장점**: 손실 함수로 역전파 가능, 빠른 최적화
- **관계**: PyTorch 같은 자동 미분 프레임워크와 결합되어 엔드-투-엔드 학습 가능

## 8. 결론 및 시사점

### 논문의 기여

1. **새로운 프레임워크**: 자율주행 장면의 동적 재구성을 위한 체계적인 접근
   - 정적 배경과 동적 객체를 분리 처리
   - 시간 축을 고려한 누적적 구성

2. **강력한 기하 정보 활용**: LiDAR 프라이어 통합으로 신뢰도 증대
   - 단순 이미지 기반 방법 대비 PSNR 8% 향상
   - SSIM 13.8% 향상으로 구조적 정확도 확보

3. **우수한 실험 결과**: nuScenes와 KITTI-360에서 SOTA 달성
   - 멀티카메라 일관성
   - 동적 객체 고품질 재구성
   - 코너 케이스 시뮬레이션 가능

### 실무적 시사점

**자율주행 안전성**
- 고품질 장면 재구성으로 시뮬레이션 환경 개선
- 극한 상황(고속 움직임, 불완전한 관찰)에서도 작동
- 시스템의 신뢰도 및 안정성 향상

**확장성과 효율성**
- 대규모 장면 처리 가능 (메모리 효율적)
- 렌더링 속도 우수 (실시간 처리 가능)
- 다양한 센서 조합 지원

**기술 발전 방향**
- 다중 센서 융합의 중요성 확인 (비전 + LiDAR)
- 정적-동적 분리의 효율성 입증
- 신경망 표현(NeRF, GS) vs 기하 기반 방법의 보완 관계 제시

### 한계 및 향후 연구

**현재 한계**
- 매우 작고 반사적인 표면(거울, 유리) 처리 어려움
- 고밀도 가우시안 최적화로 인한 높은 계산 비용

**향후 개선 방향**
- 더 나은 반사/굴절 모델링
- 적응적 가우시안 밀도 조절
- 실시간 처리를 위한 가속화 기법
