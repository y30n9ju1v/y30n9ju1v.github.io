## 1. 소개 및 배경

### 요약

CARLA는 자동주행 연구 커뮤니티의 근본적인 문제를 해결하기 위해 개발되었습니다. 자동주행 시스템의 개발과 검증에는 다음과 같은 어려움이 있습니다:

- **인프라 비용**: 실제 도로에서의 테스트는 매우 비용이 많이 들고 위험합니다
- **데이터 부족**: 다양한 시나리오와 날씨 조건에서의 고품질 데이터를 수집하기 어렵습니다
- **재현성 문제**: 동일한 조건에서 반복 테스트가 불가능합니다
- **복잡성**: 실제 도시 환경의 복잡한 상황(보행자, 교통 규칙, 다양한 장애물)을 재현해야 합니다

기존의 상용 시뮬레이터들(Grand Theft Auto, TORCS 등)은 높은 시각적 충실도를 제공하지만, 정상적인 차량 모델, 상세한 센서 사양, 유연한 환경 설정이 부족했습니다.

CARLA는 이러한 문제를 해결하기 위해 다음과 같은 특징을 제공합니다:

- **오픈소스**: 누구나 접근 가능하고 확장 가능한 코드
- **현실적인 도시 환경**: 두 개의 상세한 도시 맵
- **유연한 센서 설정**: RGB 카메라, 깊이 센서, 의미론적 분할, LiDAR 등
- **다양한 날씨와 조명 조건**
- **서버-클라이언트 구조**: Python 기반의 유연한 제어 인터페이스

### 핵심 개념

- **센서 모션 제어 문제**: 3D 환경에서 자동주행 시스템을 제어하는 것의 기술적 어려움
- **시뮬레이션의 역할**: 자동주행 시스템 개발의 안전성, 재현성, 확장성을 제공
- **학습 기반 접근**: 데이터 주도적 기계학습 방식으로 자동주행 정책을 학습
- **벤치마킹**: 다양한 방식의 자동주행 시스템을 공정하게 비교 평가할 수 있는 플랫폼

## 2. 시뮬레이션 엔진

### 요약

CARLA는 Unreal Engine 4(UE4)를 기반으로 구축되었으며, 다음과 같은 핵심 컴포넌트로 구성됩니다:

#### 2.1 아키텍처

CARLA는 **서버-클라이언트** 구조를 채택합니다:
- **서버**: UE4 기반의 시뮬레이션 엔진으로, 세계의 상태를 유지하고 렌더링
- **클라이언트**: Python으로 구현된 인터페이스로, 에이전트 제어 및 데이터 수집

이 구조는 다음과 같은 장점을 제공합니다:
- 유연한 제어 및 확장성
- 신경망이나 다른 기계학습 알고리즘을 쉽게 통합 가능
- 멀티 에이전트 시뮬레이션 지원

#### 2.2 명령 및 제어 (Commands & Meta-Commands)

**명령어 (Commands)**: 차량 제어
- **Steering**: -1 (좌회전) ~ +1 (우회전)
- **Throttle**: 0 ~ 1 (엔진 출력)
- **Brake**: 0 ~ 1 (제동력)
- **Hand Brake**: 불린 (핸드브레이크 활성화)
- **Reverse Gear**: 불린 (후진 기어)

**메타명령어 (Meta-Commands)**: 환경 및 에이전트 설정 변경
- 도시 내 차량과 보행자 수 조절
- 날씨 및 조명 조건 설정
- 카메라 설정 및 센서 구성

#### 2.3 환경 (Environment)

CARLA는 다음과 같은 요소로 구성된 3D 도시 환경을 제공합니다:

**정적 객체**:
- 도로 및 인프라 (약 20km의 도로)
- 건물 (30개 이상)
- 가로수, 포장도로, 인도 등

**동적 객체**:
- 차량 (다양한 모델, 비플레이어 제어 가능)
- 보행자 (AI 제어, 길 찾기 및 상호작용)

**특별 기능**:
- **보행자 네비게이션**: 도시 지도 기반의 자동 길 찾기
- **도로 규칙 준수**: 신호등, 속도 제한, 교차로 규칙
- **동적 객체 생성**: 시뮬레이션 중 객체 추가/제거 가능

#### 2.4 센서 (Sensors)

CARLA는 유연한 센서 설정을 지원합니다:

**RGB 카메라**:
- 위치, 방향, 화각 설정 가능
- 해상도 조절 가능

**깊이 센서 (Pseudo-Sensors)**:
- 지상 진실 깊이 (Ground-Truth Depth)
- 의미론적 분할 (Semantic Segmentation): 픽셀별 객체 클래스 분류
  - 도로, 차선 표시, 신호등, 보행자, 차량, 건물 등 12개 클래스

**측정 센서**:
- 차량 위치, 방향, 속도
- 가속도, 충돌 감지
- 교통 신호등 상태
- 다른 에이전트의 위치 및 경계 박스

### 핵심 개념

- **서버-클라이언트 패러다임**: 시뮬레이션 엔진과 제어 로직의 분리
- **센서 추상화**: 다양한 센서를 유연하게 구성 및 교체 가능
- **의미론적 분할**: 픽셀별 의미 정보로 높은 수준의 인지 가능
- **높은 충실도**: 현실적인 물리 시뮬레이션과 시각적 품질
- **확장성**: 새로운 센서, 환경, 객체 타입을 쉽게 추가 가능

## 3. 자동주행 시스템 개발

### 요약

CARLA를 이용하여 세 가지 자동주행 방식을 구현하고 평가합니다:

#### 3.1 모듈식 파이프라인 (Modular Pipeline)

**구조**: 인지(Perception) → 계획(Planning) → 제어(Control)의 전통적인 자동주행 파이프라인

**구성요소**:

1. **인지 (Perception)**
   - **의미론적 분할**: 깊이 센서의 의미론적 분할 출력을 사용하여 도로, 차선, 동적 객체를 감지
   - 네트워크 구조: ResNet 기반 의미론적 분할 네트워크
   - 차선 감지: 도로 영역을 기반으로 좌측과 우측 차선 계산
   - 상호작용 감지: 보행자, 차량, 정적 객체의 근접성 판단 (AlexNet 분류기 사용)

2. **계획 (Planning)**
   - **규칙 기반 상태 머신**: 도시 환경을 위한 단순한 행동 정책
   - 상태: (1) 도로 따르기, (2) 좌회전, (3) 우회전, (4) 상호작용 회피
   - **전역 계획**: 목표 지점까지의 경로를 미리 계획 (A* 알고리즘)
   - **지역 계획**: GPS 기반 웨이포인트 생성으로 스티어링 각도 결정
   - **비례-적분-미분 제어기 (PID)**: 목표 스티어링 각도로 차량을 제어

3. **제어 (Control)**
   - 비례-적분-미분(PID) 제어기로 스티어링, 스로틀, 제동 제어
   - 현재 속도, 목표 속도, 차량 상태를 입력으로 받음

**특징**: 해석 가능하고 규칙 기반이지만, 복잡한 시나리오에서는 제한적

#### 3.2 모방 학습 (Imitation Learning - IL)

**개념**: 인간 운전자의 주행 데이터로부터 운전 정책을 직접 학습

**데이터 수집**:
- **전문가 데이터**: 인간 운전자의 주행 기록 (차선, 신호, 속도, 가속도, 충돌 등)
- **4가지 명령**: 좌회전, 우회전, 직진, 목표 도달 (도시 지도 기반)
- **데이터 증강**: 네트워크의 일반화 능력을 높이기 위해 노이즈 주입

**학습 방식**:
- 신경망이 카메라 이미지 → 운전 제어(스티어링, 스로틀, 제동) 직접 학습
- 네트워크: 4개 모듈로 구성
  - 인지 모듈: 이미지 특징 추출
  - 측정 모듈: GPS, 속도, 거리 등의 측정값
  - 조인트 입력: 두 정보 결합
  - 제어 모듈: 최종 제어 신호 생성

**네트워크 구조**:
- 입력: 200×88×3 이미지 + 128차원 측정 벡터
- Perception 모듈: Conv-Conv-Conv-FC layers
- Measurement 모듈: FC layers  
- 출력: 256차원 제어 신호

**특징**: 빠른 학습과 좋은 성능, 하지만 분포 외(out-of-distribution) 상황에서 취약

#### 3.3 강화학습 (Reinforcement Learning - RL)

**개념**: 환경과의 상호작용을 통해 보상 신호를 최대화하는 정책을 학습

**학습 방식**:
- **Actor-Critic (A3C) 알고리즘**: 비동기적 병렬 학습
- **보상 함수**:

$$r_t = 1000(d_{t-1} - d_t) + 0.05(v_t - v_{t-1}) - 0.0002(c_t - c_{t-1}) - 2(s_t - s_{t-1}) - 2(o_t - o_{t-1})$$

**수식 설명**:
- **$r_t$**: t 시점의 총 보상 (학습의 목표)
- **$1000(d_{t-1} - d_t)$**: 목표까지의 거리 감소에 대한 강한 보상
  - $d_t$: 현재 목표까지의 거리 (km)
  - 목표에 가까워질수록 높은 보상
- **$0.05(v_t - v_{t-1})$**: 속도 증가에 대한 약한 보상
  - 안정적이고 빠른 주행 유도
- **$-0.0002(c_t - c_{t-1})$**: 충돌 패널티
  - 충돌이 발생하면 큰 페널티
- **$-2(s_t - s_{t-1})$**: 인도 침범 패널티
  - 차량의 밑바닥이 인도와 겹치는 것을 방지
- **$-2(o_t - o_{t-1})$**: 반대 차선 침범 패널티
  - 정상적인 차선 유지 강제

**학습 세부사항**:
- 10개 병렬 스레드에서 동시 학습
- 10M 시뮬레이션 스텝 학습 (약 12일의 실제 주행에 해당)
- 네트워크는 이미지와 측정값을 입력으로 정책을 학습

**특징**: 복잡한 시나리오에 적응 가능하지만, 학습 시간이 오래 걸림

### 핵심 개념

- **모듈식 파이프라인**: 각 단계가 독립적이고 해석 가능한 구조
- **모방 학습 (IL)**: 인간 운전자의 예시(demonstration)로부터 직접 학습
- **강화학습 (RL)**: 시뮬레이터와의 상호작용을 통해 보상을 최대화하는 정책 학습
- **행동 모방**: 특정 지시(좌회전, 우회전 등)에 따라 행동하도록 강제
- **분포 시프트 (Distribution Shift)**: IL이 학습 데이터와 다른 상황에서 성능 저하

## 4. 실험 및 결과

### 요약

CARLA의 세 가지 접근 방식의 성능을 비교하기 위해 체계적인 실험을 수행했습니다.

#### 4.1 실험 설정

**테스트 환경**:
- **Town 1** (학습 환경): 도시 지도 1
- **Town 2** (테스트 환경): 도시 지도 2 (학습에서 미사용)

**테스트 시나리오** (난이도 순):
1. **Straight**: 목표가 시작점 정면 200m (Town 1) / 100m (Town 2)
2. **One turn**: 한 번의 교차로 통과, 목표 거리 400m (Town 1) / 170m (Town 2)
3. **Navigation**: 교차로와 동적 객체 존재, 목표 거리 770m (Town 1) / 560m (Town 2)
4. **Nav. dynamic**: Navigation과 동일하지만 동적 객체(차량, 보행자) 추가

**날씨 조건** (학습 조건과의 일반화 평가):
- **Training Weather Set**: 학습 중 사용한 날씨
- **New Weather Set**: 새로운 날씨 (학습에 미포함, 일반화 능력 평가)

각 태스크에서 에피소드는 최대 120초 동안 실행되며, 목표 지점에 도달하면 성공으로 간주됩니다.

#### 4.2 평가 메트릭

**성공률**: 목표 지점에 성공적으로 도달한 에피소드의 비율

**평균 이동 거리**: 두 교차로 사이에서 이동한 평균 거리 (km)
- 높은 값은 안정적인 주행을 의미
- 중앙선 침범, 인도 침범 등의 위반을 통해 얼마나 효율적으로 이동했는지 측정

**실패 유형**:
- **충돌**: 차량이나 정적 객체와의 충돌
- **인도 침범**: 인도로 진입
- **반대 차선 침범**: 대향 차선으로 진출

#### 4.3 실험 결과

**주요 발견**:

1. **직선 주행 (Straight)**: 모든 방식이 높은 성공률
   - MP: 98% (학습) → 96% (테스트)
   - IL: 89% (학습) → 89% (테스트)
   - RL: 82% (학습) → 61% (테스트)

2. **회전이 포함된 주행**: MP > IL ≈ RL
   - One turn: MP 89%, IL 79%, RL 59%
   - Navigation: MP 80%, IL 86%, RL 20%

3. **동적 객체와 함께 (Nav. dynamic)**:
   - MP: 77%, IL: 83%, RL: 7%
   - RL의 급격한 성능 하락 (시뮬레이터 특화, 일반화 능력 부족)

4. **새로운 날씨 조건**:
   - MP: 일반화 능력 우수 (새로운 날씨에도 성능 유지)
   - IL: 성능 저하 (분포 시프트 문제)
   - RL: 심각한 성능 저하 (환경 변화에 매우 취약)

5. **교차로 간 평균 거리**:
   - **Opposite lane** (반대 차선): MP 최우수
   - **Sidewalk** (인도 침범): IL 최악
   - **Collision** (충돌): RL 최악

### 핵심 개념

- **분포 외 일반화 (Out-of-Distribution Generalization)**: 학습과 다른 조건에서의 성능
- **시뮬레이터 특화**: RL이 시뮬레이터의 특정 특성에 과적합되는 경향
- **모듈식 파이프라인의 견고성**: 규칙 기반 접근이 새로운 날씨에 더 잘 대응
- **분포 시프트 (Distribution Shift)**: IL에서 학습 데이터와 다른 상황의 성능 저하
- **복잡한 시나리오의 어려움**: 동적 객체가 있는 환경에서 모든 방식이 성능 저하

## 5. 결론 및 시사점

### 요약

CARLA는 자동주행 연구를 위한 **개방적, 확장 가능한 플랫폼**을 제공합니다. 이 논문의 주요 기여는 다음과 같습니다:

1. **오픈소스 시뮬레이터**: 누구나 접근할 수 있는 도시 주행 시뮬레이션 플랫폼
   - 높은 시각적 품질
   - 현실적인 물리 시뮬레이션
   - 유연한 센서 구성

2. **벤치마킹 플랫폼**: 자동주행 방식을 공정하게 비교할 수 있는 표준화된 평가 환경
   - 다양한 시나리오 (직진, 회전, 네비게이션, 동적 객체)
   - 일관된 메트릭 정의

3. **실험적 통찰**:
   - **모듈식 파이프라인**: 새로운 조건으로의 일반화 능력이 우수하지만 복잡한 시나리오에서는 제한적
   - **모방 학습**: 학습 데이터와 같은 조건에서는 우수하지만 분포 시프트에 취약
   - **강화학습**: 복잡한 시나리오 학습 잠재력 있으나, 시뮬레이터 특화와 일반화 문제

### 핵심 개념

- **벤치마킹의 중요성**: 다양한 방식을 객관적으로 비교 가능
- **일반화 능력**: 새로운 환경과 조건에 대한 적응력의 중요성
- **하이브리드 접근**: 각 방식의 장점을 결합한 시스템 개발의 필요성
- **시뮬레이션-현실 격차**: 시뮬레이터에서 학습한 정책이 현실에서도 작동하도록 하기 위한 도메인 적응 필요
- **오픈소스의 가치**: 공개 플랫폼을 통한 커뮤니티 협력의 중요성

## 핵심 개념 정리

### 자동주행 시스템의 세 가지 접근법

| 접근법 | 특징 | 장점 | 단점 |
|--------|------|------|------|
| **모듈식 파이프라인 (MP)** | 인지→계획→제어 | 해석 가능, 새로운 환경 일반화 우수 | 복잡한 시나리오 제한 |
| **모방 학습 (IL)** | 전문가 데이터 학습 | 간단한 구현, 학습 빠름 | 분포 시프트, 새로운 조건 취약 |
| **강화학습 (RL)** | 보상 신호 최대화 | 복잡한 시나리오 학습 잠재력 | 학습 시간 오래, 일반화 취약 |

### 센서 및 측정

**의미론적 분할**: 픽셀별로 객체의 클래스를 분류
- 도로, 차선, 신호등, 보행자, 차량 등을 색깔로 구분
- 컴퓨터 비전 기반 자동주행에 필수적인 정보

**깊이 센서**: 3D 거리 정보 제공
- LiDAR나 스테레오 비전의 대안
- 카메라와 달리 조명 변화에 영향 적음

### 자동주행의 도전 과제

1. **분포 시프트**: 학습과 테스트 데이터의 분포 차이
2. **시뮬레이션-현실 격차**: 시뮬레이터와 현실의 차이
3. **일반화 능력**: 새로운 환경, 날씨, 도로 조건에 대한 적응
4. **안전성 보장**: 자동주행 시스템의 신뢰성과 안전성 확보
5. **계산 효율성**: 실시간 처리를 위한 최적화

CARLA는 **자동주행 연구의 민주화**를 이룬 중요한 기여입니다. 이 논문을 통해 시뮬레이션의 중요성, 다양한 자동주행 방식의 강점과 약점, 그리고 앞으로의 연구 방향을 제시합니다.

4. **오픈소스 협력**: 공개 플랫폼을 통한 커뮤니티 전체의 발전

CARLA는 출시 이후 자동주행 연구 커뮤니티의 표준 플랫폼이 되었으며, 수많은 논문과 프로젝트에서 활용되고 있습니다.
