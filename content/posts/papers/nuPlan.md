## Chapter 1: Introduction

**요약**

자율주행 ML 연구에서 기존 벤치마크들은 **단기 모션 예측(prediction)** 에 초점을 맞추었다. 즉, "다른 차량이 다음 3초 동안 어디로 움직일 것인가"를 예측하는 태스크다. 그러나 실제 자율주행에 필요한 것은 **계획(planning)** — "자차(ego vehicle)가 어떻게 행동해야 하는가" — 이다.

논문은 기존 예측 벤치마크의 세 가지 핵심 한계를 지적한다 (Figure 1):

1. **목표 모호성 (Fig. 1a)**: 교차로에서 좌회전/우회전 모두 유효하지만 데이터에는 하나의 선택만 있음. L2 거리로 평가하면 다른 선택지는 무조건 나쁜 것으로 평가됨
2. **다중 모달 무시 (Fig. 1b)**: 즉시 합류하거나 나중에 합류하는 것 모두 안전하지만, L2 메트릭은 이를 구분하지 못함
3. **에이전트 상호작용 부재 (Fig. 1c)**: 차선 변경 결정은 주변 모든 차량의 연속적 행동을 고려해야 하지만, 오픈루프에서는 상호작용이 없음

이를 해결하기 위해 nuPlan은:
- 대규모 실제 주행 데이터셋 (1500시간, 4개 도시)
- **반응형(reactive) 에이전트**를 포함한 클로즈드루프 시뮬레이터
- 교통 법규 위반, 인간 주행 유사도, 차량 동역학, 목표 달성 등 **계획 특화 지표**

를 함께 제공한다.

**핵심 개념**

- **예측(Prediction) vs 계획(Planning)**: 예측은 타 에이전트의 미래 궤적을 추정하는 것. 계획은 자차가 취할 행동을 결정하는 것. 계획은 예측보다 장기적이고 목표 지향적
- **오픈루프 vs 클로즈드루프**: 오픈루프는 고정된 데이터로 평가하여 에이전트 행동이 환경에 영향 없음. 클로즈드루프는 시뮬레이터에서 자차 행동이 다른 에이전트에 반응을 유발
- **분포 이탈(Distribution Shift)**: 오픈루프 평가에서 좋은 성능을 내도 실제 주행에서는 오류가 누적되어 좋지 않은 결과를 낼 수 있는 현상

## Chapter 2: Related Work

**요약**

**예측 데이터셋**: Argoverse, nuScenes, Waymo Open Motion Dataset 등이 있으나 모두 단기 예측(3~8초) 중심이며 오픈루프 평가만 지원.

**시뮬레이터**: CARLA(그래픽 렌더링 기반, 도메인 갭 존재), AirSim(고품질 시각 시뮬레이션), nuPlan(실제 데이터 기반 클로즈드루프, 도메인 갭 최소화).

**ML 기반 계획**: ChauffeurNet(모방 학습), MP3(라이다 기반 맵리스 계획), 다양한 멀티모달 접근 등이 있으나 공통 벤치마크 부재로 비교 어려움. nuPlan이 이 공백을 채움.

## Chapter 3: Dataset

**요약**

### 규모와 구성

- **총 1500시간** 실제 주행 데이터
- **4개 도시**: 라스베이거스(복잡한 교차로, PUDOs), 보스턴(이중 주차 차선, 독특한 교통 패턴), 피츠버그(자체 교통 우선권 패턴), 싱가포르(좌측 통행)
- **센서**: LiDAR 포인트 클라우드, 카메라 이미지, 지역화 정보, 조향 입력
- **주석**: 전체 데이터셋에 자동 레이블링 적용 (오프라인 인식 시스템 사용)
- 공개 서브셋 약 200+ TB

### 자동 레이블링

PointPillars + CenterPoint, 멀티뷰 퓨전(MVF++)을 사용하여 오프라인으로 고정밀 추적 레이블을 생성. 실시간 제약 없이 고품질 어노테이션 달성.

### 시나리오 태깅

자동으로 복잡한 운전 시나리오에 태그를 부여:
- 차선 변경, 좌/우회전(신호 있음/없음)
- 교차로에서 보행자·자전거와의 상호작용
- 근접 주행, 급가속, 공사 구간 통과 등

이 태그는 시나리오 기반 평가 지표 계산에 활용된다.

## Chapter 4: Benchmarks

**요약**

### 4.1 태스크 정의

플래너는 특정 타임스텝에서 **계획된 위치와 방향(planned trajectory)**을 반환한다. 제공된 컨트롤러가 이 궤적을 추종하며 차량을 구동한다.

세 가지 난이도의 태스크:

**1. 오픈루프 (Open-loop)**
- 플래너가 인간 운전자를 모방하도록 학습
- 매 타임스텝마다 계획 궤적을 사전 정의된 지표로 평가
- 차량을 실제로 제어하지 않으므로 에이전트 상호작용 없음
- 쉽지만 분포 이탈 문제 존재

**2. 비반응형 클로즈드루프 (Non-Reactive Closed-loop)**
- 플래너가 계획한 궤적으로 차량을 실제 제어
- 다른 에이전트는 기록된 궤적을 그대로 재현 (반응 없음)
- 비반응형이므로 대부분의 클로즈드루프 개입은 차량 충돌로 나타남 (NAVSIM의 이론적 기반)

**3. 반응형 클로즈드루프 (Reactive Closed-loop)**
- 플래너가 계획한 궤적으로 차량을 실제 제어
- **다른 모든 에이전트도 IDM(Intelligent Driver Model) 등으로 반응**
- 가장 현실적이지만 계산 비용이 높음

### 4.2 평가 지표

**공통 지표 (Common Metrics)**:

| 지표 | 설명 |
|------|------|
| **교통 법규 위반** | 다른 에이전트와의 충돌, 차선 이탈 궤적, TTC, 상대 속도, 통과 속도 |
| **인간 주행 유사도** | 종방향 속도 오차, 정지 위치 오차, 횡방향 위치 오차, 저크/가속도 비교 |
| **차량 동역학** | 주행 편안함. 저크, 가속도, 조향, 차량 진동 등으로 측정 |
| **목표 달성 (Goal Achievement)** | L2 거리로 측정한 목표 웨이포인트 접근도 |

**시나리오 기반 지표 (Scenario-based Metrics)**:
- **차선 변경**: TTC, 목표 차선까지의 시간 간격
- **보행자/자전거 상호작용**: 통과 시 상대 위치 변화
- **교차로 결정**: 횡단보도·비신호 교차로에서 플래너와 인간의 결정 일치 여부

### 4.3 CLS (Closed-Loop Score) 구성

nuPlan의 클로즈드루프 점수는 위 지표들을 가중 결합하여 계산한다:

$$\text{CLS} = \frac{1}{N} \sum_{i=1}^{N} \left( \prod_{m \in \text{penalties}} s_m^{(i)} \right) \times \left( \sum_{k \in \text{weighted}} w_k \cdot s_k^{(i)} \right)$$

**수식 설명**:
- **$N$**: 전체 평가 시나리오 수
- **$s_m^{(i)}$**: i번째 시나리오에서 패널티 지표 $m$의 점수 (충돌 시 0 또는 감점 곱)
- **$w_k$**: 가중 평균 지표 $k$의 가중치
- **$s_k^{(i)}$**: i번째 시나리오에서 가중 지표 $k$의 점수
- NAVSIM의 PDMS 점수가 이 CLS 구조를 직접 계승하여 설계됨

### 4.4 벤치마크 챌린지

평가 서버에 플래너 코드를 제출하면 비공개 테스트 셋에서 자동으로 클로즈드루프 평가가 수행된다. NeurIPS 2021에서 데이터셋 공개 및 첫 챌린지 개최.

## 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **클로즈드루프 평가** | 플래너가 실제로 차량을 제어하며 환경과 상호작용. 오류가 누적되는 현실과 가장 유사한 평가 방식 |
| **반응형 vs 비반응형** | 반응형: 다른 차량이 자차 행동에 반응. 비반응형: 다른 차량은 기록된 궤적만 재현 (NAVSIM이 후자 채택) |
| **목표 모호성** | 교차로에서 여러 경로가 모두 유효할 때, 인간이 선택한 경로만 정답으로 삼는 것의 한계 |
| **시나리오 기반 지표** | 특정 상황(차선 변경, 보행자 교차 등)에 특화된 평가 지표. 일반 지표로 놓치는 세부 능력 평가 |
| **자동 레이블링** | 오프라인 인식 시스템으로 대규모 데이터를 고품질로 자동 주석 처리. 인간 레이블러 없이 1500시간 데이터 구축 가능 |
| **IDM (Intelligent Driver Model)** | 앞차와의 간격, 속도를 수식으로 모델링한 고전적 차량 추종 모델. 반응형 시뮬레이션에서 배경 차량 행동에 사용 |
| **분포 이탈 (Distribution Shift)** | 학습 데이터와 다른 상황에 처했을 때 성능이 급격히 떨어지는 현상. 클로즈드루프 평가가 이를 드러냄 |

## 결론 및 시사점

nuPlan은 자율주행 ML 계획 연구에 필요한 세 가지를 한꺼번에 제공한다:

1. **대규모 현실 데이터**: 1500시간, 4개 도시, 다양한 날씨·교통 조건 → 일반화 가능한 플래너 개발 가능
2. **클로즈드루프 평가**: 오픈루프의 분포 이탈 문제 극복. 반응형/비반응형 두 설정 모두 지원
3. **계획 특화 지표**: 교통 법규, 편안함, 목표 달성, 시나리오별 지표 → 예측 지표(ADE/FDE)의 한계 극복

**후속 영향**:
- **NAVSIM**이 nuPlan의 OpenScene 데이터를 재배포하고 CLS 구조를 계승한 PDMS를 정의함
- **PDM-Closed** 등 nuPlan 전용 규칙 기반 플래너들이 개발되어 ML 방법의 baseline이 됨
- UniAD, VAD 등 E2E 방법들이 nuPlan에서의 평가를 추가하여 계획 능력을 검증

**한계**:
- 비반응형 설정에서 대부분의 개입이 단순 충돌로 나타나 장기 상호작용 평가 어려움
- 200+ TB 전체 데이터셋 사용이 현실적으로 어려움 → OpenScene(2TB→450GB)으로 축소 재배포
- 초기 지표 정의가 커뮤니티 피드백을 반영한 개선 필요 (이후 nuPlan 1.1, 1.2 등으로 업데이트)
